TypeScript - Entenda os tipos no TypeScript - Parte 1

Os tipos, aqui começa uma das maiores diferenças do ts para o js, a tipagem. O TypeScript oferece todos os recursos do JavaScript e uma camada adicional em cima deles: o sistema de tipos do TypeScript. O JavaScript fornece tipos primitivos de linguagem como string e número, mas não verifica se atribuímos eles de forma correta. O TypeScript sim vai fazer essa verificação.

TypeScript - Sendo mais produtivo com a verificação de tipo estático

O grande objetivo do TypeScript é ser um verificador de tipo estático para programas JavaScript - em outras palavras, o sistema interpreta o código antes dele ser executado e garante que os tipos do código estejam corretos. Então, ele nos ajuda a identificar possíveis erros ou problemas no nosso código conforme escrevemos no VS Code, então antes mesmo de executar esse código.

Desabilitar tipagem com tipo Any ✔️

TypeScript - Desabilitar tipagem com tipo Any

O TypeScript também tem um tipo especial, que podemos usar quando não queremos que um valor específico cause erros de verificação de tipo, ou quando não temos certeza do tipo, é o tipo any, que traduzido pra português significa qualquer.
Então, se falamos que o elemento seria de qualquer tipo, mas é importante tentar evitar ao máximo o uso do any e vou te mostrar por que.

Mas quando usar o any então?
- Quando estamos trabalhando com código legado que não pode ser facilmente refatorado para usar tipos específicos.
- Quando estamos trabalhando com bibliotecas externas cujos tipos não são definidos ou não estão disponíveis.

Em resumo, o tipo any é uma ferramenta poderosa, mas deve ser usado com cautela.
Sempre que possível, especifique o tipo das variáveis para ter a segurança de tipos que o TypeScript oferece e para tornar o seu código mais fácil de manter e entender.

Propriedades e parâmetros opcionais ✔️

TypeScript - Propriedades e parâmetros opcionais

No TypeScript, podemos definir propriedades e parâmetros com valores opcionais, por exemplo:

Código do tipo do Customer com parâmetro opcional
            
                type Customer = {
                    firstName: string;
                    lastName?: string;
                    age: number;
                 }
Código com a criação de um novo Customer sem o parâmetro lastName
            
                const newCustomer: Customer = {
                    firstName: 'João',
                    age: 34
                 }
                 
                 console.log(newCustomer)
Exemplo de função com parâmetro opcional
            
                function printName(first: string, last?: string) {
                    console.log(`O primeiro nome é : ${first}`)
                    console.log(`O ultimo nome é : ${last}"`)
                 }
                 
                 // agora eu posso chamar a função passando só um argumento pq o outro é opcional
                 printName("Ricardo");
                 
                 // mas se eu quiser passar os dois eu posso também
                 printName("Ricardo", "Silva");
