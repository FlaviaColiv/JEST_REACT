Componente Stateless vs Componente Statefull

React Avançado - Componente Stateless vs Componente Statefull

Nesta aula sobre React Avançado, você aprenderá sobre a diferença entre Componente Stateless (sem estado) e Componente Stateful (com estado).
Será explicado o conceito de Componente Stateless, que é um componente React que não possui um estado interno. Esses componentes são mais simples e focados na apresentação de informações, sendo úteis para renderizar dados estáticos ou receber props e exibir conteúdo na interface.
Por outro lado, você também aprenderá sobre Componentes Stateful, que têm um estado interno e podem lidar com alterações dinâmicas ao longo do tempo. Esses componentes são responsáveis por gerenciar dados e responder a eventos, permitindo a interatividade na aplicação React. Serão abordados conceitos como a criação de um estado, a atualização do estado e a renderização condicional com base no estado atual.

Introdução aos Hooks

React Avançado - Introdução aos Hooks

Nesta aula introdutória, você será introduzido aos Hooks, uma poderosa funcionalidade do React que permite a você utilizar estado e outras características do React em componentes funcionais.
Primeiramente, você aprenderá o que são Hooks e por que eles foram introduzidos no React. Os Hooks são uma forma de reutilizar lógica de estado e efeitos colaterais em componentes funcionais, eliminando a necessidade de escrever componentes de classe. Você entenderá as vantagens dos Hooks em relação aos componentes de classe tradicionais.
Link de apoio: Built-in React Hooks – React

Se você está familiarizado com os métodos do ciclo de vida do React, você pode pensar no Hook userEffect como componentDidMount, componentDidUpdate e componentWillUnmount combinados.

Regras:
- Não chame Hooks dentro de loops, condições ou funções aninhadas.
- Apenas chame Hook de componentes funcionais. Não chame Hooks de funçoes JS.

A melhor forma de Criar Formulários no ReactJS com React Hook Form

React Avançado - A melhor forma de Criar Formulários no ReactJS com React Hook Form

Descubra a melhor maneira de criar formulários em ReactJS usando React Hook Form. Esta aula irá tornar o processo de criação de formulários mais simples e eficiente.
Link de apoio: React Hook Form - performant, flexible and extensible form library (react-hook-form.com)

Para lidar com a adição da classe campo-preenchido apenas quando o campo é validado e não possui erros, usamos a propriedade formState.touchedFields do react-hook-form para verificar se o campo foi tocado. Além disso, combinamos isso com a verificação de erros para aplicar a classe campo-preenchido de forma condicional.
No código, a classe campo-preenchido será aplicada apenas se o campo foi tocado (touchedFields.campo) e não contém erros (!errors.campo). Isso garantirá que a classe só seja adicionada quando apropriado, evitando que seja aplicada ao inicializar o formulário.
